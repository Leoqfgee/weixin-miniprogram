## 3. 详细测试记录

### 3.1 卖家流程

- 测试步骤：卖家登录 -> 我的 -> 我发布的 -> 我卖出的 -> 对待发货订单执行发货 -> 选择校内面交 -> 填写地点 -> 提交。
- 实际结果：“我发布的”和“我卖出的”均正常加载；空地点提交返回“校内面交需填写地点或说明”；填写“图书馆东门”后发货成功，订单变为待收货。
- 发现问题：卖出列表在小屏模拟器中部分按钮文字换行较多，但仍可操作；历史演示数据存在标题乱码和异常价格。
- 相关接口或文件：`POST /api/v1/deliveries/{order_id}/seller-deliver`，`miniprogram/pages/order/sold/`，`miniprogram/pages/order/detail/`。
- 修复情况：校验属于预期行为，不修改；列表换行和历史数据问题记录为体验/数据治理项。

### 3.2 买家流程

- 测试步骤：买家登录 -> 我买到的 -> 打开待收货订单 -> 确认收货 -> 打开订单详情 -> 评价 -> 填写评价内容并发布。
- 实际结果：确认收货成功，订单进入待评价；评价提交成功，订单详情展示 5 星评价；卖家端最终显示“双方完成”。
- 发现问题：订单列表中的操作入口较依赖用户理解状态，待评价按钮主要在订单详情中出现。
- 相关接口或文件：`POST /api/v1/deliveries/{order_id}/buyer-confirm`，`POST /api/v1/reviews`，`miniprogram/pages/order/bought/`，`miniprogram/pages/review/apply/`。
- 修复情况：核心流程可用，不修改。

### 3.3 商品、筛选与收藏

- 测试步骤：首页 -> 筛选商品 -> 检查分类、成色、排序、校区、价格、日期控件 -> 打开商品详情 -> 检查收藏、联系卖家、立即购买 -> 打开我的收藏。
- 实际结果：最低价、最高价、开始日期、结束日期控件真实存在；收藏列表可加载；商品详情“联系卖家”可直接进入对应商品聊天页。
- 发现问题：部分旧商品图片使用失效地址；部分旧数据标题乱码；演示商品存在不合理高价。
- 相关接口或文件：`GET /api/v1/products`，`miniprogram/pages/category/index.*`，`miniprogram/pages/favorite/`，`miniprogram/pages/product/detail/`。
- 修复情况：筛选与收藏无需修复；历史数据问题未修改业务代码。

### 3.4 消息列表与私聊

- 测试步骤：点击消息 Tab -> 点击会话 -> 进入聊天页 -> 点击文本输入框 -> 输入并发送“CU测试消息” -> 返回商品详情并点击“联系卖家”再次进入聊天。
- 实际结果：消息列表和私聊页均可进入；商品详情私聊入口真实有效；修复后文本可输入并成功发送，消息出现在聊天记录中。
- 发现问题：修复前聊天输入框在模拟器上被三个按钮的微信默认外边距挤压到几乎零宽，无法稳定点击输入。
- 相关接口或文件：`GET /api/v1/messages/conversations`，`GET /api/v1/messages/conversations/{conversation_id}`，`POST /api/v1/messages`，`miniprogram/pages/message/chat/index.wxss`。
- 修复情况：已做最小 WXSS 修复，并使用 Computer Use 重新输入、发送消息验证通过。

### 3.5 个人中心、资料与地址

- 测试步骤：打开个人中心 -> 编辑资料 -> 检查头像、昵称、校区、简介 -> 返回并打开收货地址 -> 检查地址列表和操作入口。
- 实际结果：个人资料编辑页存在；头像和昵称可编辑；地址列表、默认地址、新增、编辑、删除入口均可加载。
- 发现问题：页面首次导航和数据渲染通常需要 1.5 至 3 秒。
- 相关接口或文件：`miniprogram/pages/profile/edit/`，`miniprogram/pages/address/`，`backend/app/blueprints/addresses.py`。
- 修复情况：功能可用，不修改。

### 3.6 管理员流程

- 测试步骤：管理员登录 -> 商品审核 -> 申诉仲裁 -> 统计报表 -> 后台日志。
- 实际结果：商品审核显示待审核商品和通过/驳回入口；申诉仲裁可加载空状态；统计报表展示基础指标；后台日志展示基础统计和操作日志。
- 发现问题：统计报表信息维度不足，不能完整支撑课程验收中的运营分析；申诉仲裁当前无测试数据，未执行真实仲裁。
- 相关接口或文件：`GET /api/v1/admin/products`，`GET /api/v1/admin/appeals`，`GET /api/v1/admin/stats`，`GET /api/v1/admin/operation-logs`，`miniprogram/pages/admin/`。
- 修复情况：基础功能可用；报表完整性列入后续开发。

## 4. 已修复问题

### 聊天页文本输入框几乎不可用

- 问题描述：聊天页可进入、历史消息可显示，但底部文本输入框被操作按钮挤压到几乎零宽，Computer Use 多次点击无法稳定聚焦。
- 根因：微信小程序 `button` 存在默认外边距；聊天输入框与多个按钮处于同一个 flex 容器中，按钮默认外边距占用横向空间，输入框又缺少最小宽度。
- 修改文件：`miniprogram/pages/message/chat/index.wxss`
- 修改内容：为文本输入/语音按钮增加 `min-width: 120rpx`；为图标按钮和发送按钮增加 `margin: 0`、`flex: none`。
- 复测结果：热更新后输入框正常显示“输入消息”；Computer Use 成功输入并发送“CU测试消息”，消息在聊天记录中显示，复测通过。

- 私聊是否没有真正进入聊天界面：不存在该问题。商品详情“联系卖家”已实际进入聊天界面。

- 个人主页/头像昵称编辑是否缺失：不缺失。个人主页入口和资料编辑页均存在，头像、昵称、校区、简介可编辑。
