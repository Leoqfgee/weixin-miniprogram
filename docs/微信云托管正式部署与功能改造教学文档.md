# 微信云托管正式部署与功能改造教学文档

本文档面向第一次部署本项目的同学，说明如何把“校园二手交易平台”部署为：

微信小程序前端 -> 微信云托管 Flask 后端 -> 微信云托管 MySQL -> 千问 API -> 微信账号登录。

## 1. 项目目录

- `miniprogram/`：微信小程序前端，使用微信开发者工具打开。
- `backend/`：Flask 后端，部署到微信云托管。
- `backend/Dockerfile`：云托管容器构建文件。
- `backend/sql/init_db.sql`：正式 MySQL 表结构初始化脚本，执行前必须备份数据库。
- `backend/scripts/init_db.py`：课程演示数据初始化脚本或接口复用逻辑。
- `docs/`：部署、测试和课程说明文档。

## 2. 云托管环境变量

进入微信云托管控制台，选择服务 `flask-fnnj`，在服务设置或发布高级设置中填写环境变量。

必须填写：

```text
DB_BACKEND=mysql
MYSQL_ADDRESS=云托管 MySQL 地址
MYSQL_PORT=3306
MYSQL_USERNAME=root
MYSQL_PASSWORD=云托管推送给你的数据库密码
MYSQL_DATABASE=flask_demo
JWT_SECRET=一串足够长的随机密钥
INIT_TOKEN=一串初始化口令
FLASK_DEBUG=0
FLASK_HOST=0.0.0.0
FLASK_PORT=80
WECHAT_AUTH_MODE=wechat
WECHAT_APPID=微信小程序 AppID
WECHAT_SECRET=微信小程序 AppSecret
AI_MODE=qwen
DASHSCOPE_API_KEY=阿里云百炼 DashScope API Key
QWEN_MODEL=qwen-plus
PAYMENT_MODE=mock
COS_BUCKET=对象存储 bucket，可暂留空
COS_REGION=对象存储地域，可暂留空
```

注意：必须写 `MYSQL_DATABASE`，不能写成 `MYSQL_DATABAS`，最后的 `E` 不能少。

所有密码、JWT 密钥、千问 API Key、微信 AppSecret 都只能写在环境变量里，不要写进代码，也不要提交到仓库。

## 3. 端口和 Dockerfile

微信云托管发布时端口填写：

```text
80
```

容器必须监听：

```text
0.0.0.0:80
```

本项目入口是 `wsgi.py` 里的 `app`，所以云托管启动命令为：

```bash
gunicorn --bind 0.0.0.0:${PORT:-80} wsgi:app
```

如果发布时仍填 5000，会导致云托管转发端口和容器监听端口不一致。

## 4. MySQL 初始化

先在 DMS 或数据库控制台备份当前数据库。

然后打开 DMS 的 SQL 窗口，执行：

```sql
source backend/sql/init_db.sql;
```

如果 DMS 不支持 `source`，就打开 `backend/sql/init_db.sql`，复制全部 SQL 到 SQL 窗口执行。

课程演示数据可以通过云托管调试接口初始化：

```text
POST /api/v1/init-demo-data
Header: X-Init-Token = 你配置的 INIT_TOKEN
Body: {}
```

该接口只在你知道初始化口令时可用。

## 5. 健康检查

浏览器访问：

```text
https://flask-fnnj-267255-4-1440900946.sh.run.tcloudbase.com/api/v1/health
```

成功标准：

```json
{
  "code": 0,
  "data": {
    "status": "ok",
    "database": {
      "backend": "mysql",
      "status": "ok",
      "error": null
    }
  }
}
```

继续验证真实数据库连接：

```text
GET /api/v1/health/db
```

该接口会执行 MySQL `SELECT 1`，不是假返回。

## 6. 小程序前端配置

小程序已在 `miniprogram/app.js` 中初始化云开发环境：

```js
wx.cloud.init({
  env: 'prod-d2g73fc4ha6d2e317',
  traceUser: true
})
```

所有普通 API 请求都通过 `miniprogram/utils/request.js` 调用：

```js
wx.cloud.callContainer({
  config: { env: 'prod-d2g73fc4ha6d2e317' },
  path: '/api/v1/...',
  header: { 'X-WX-SERVICE': 'flask-fnnj' }
})
```

不要再默认请求 `10.x.x.x:5000` 这类局域网 IP。

## 7. 微信登录验证

生产环境配置：

```text
WECHAT_AUTH_MODE=wechat
WECHAT_APPID=你的 AppID
WECHAT_SECRET=你的 AppSecret
```

验证步骤：

1. 微信开发者工具重新编译小程序。
2. 手机预览小程序。
3. 点击“微信账号登录”。
4. 后端调用微信 `jscode2session` 获取 openid。
5. 后端创建或更新用户并返回 JWT。
6. 后续请求自动携带 `Authorization: Bearer token`。

如果报“微信登录配置缺失”，检查 `WECHAT_APPID` 和 `WECHAT_SECRET`。

## 8. 千问 API 验证

生产环境配置：

```text
AI_MODE=qwen
DASHSCOPE_API_KEY=你的 DashScope API Key
QWEN_MODEL=qwen-plus
```

后端接口：

```text
POST /api/v1/ai/title
POST /api/v1/ai/polish
POST /api/v1/ai/description
```

验证步骤：

1. 登录小程序。
2. 进入发布商品页。
3. 输入商品标题或描述。
4. 点击 AI 标题建议或 AI 润色描述。
5. 成功后检查 MySQL 中 `ai_generation_logs` 或 `cloud_documents` 里的 `ai_generation_logs` 数据。

如果未配置 `DASHSCOPE_API_KEY`，接口会返回真实错误，不会假装成功。

## 9. 商品和订单验证

商品列表：

```text
GET /api/v1/products
```

商品详情：

```text
GET /api/v1/products/:id
```

发布商品：

```text
POST /api/v1/products
```

创建订单：

```text
POST /api/v1/orders
```

后端模拟支付：

```text
POST /api/v1/payments/mock-confirm
```

支付必须请求后端接口，由后端校验订单并更新 MySQL，不允许前端 `setData` 假装支付成功。


## 11. 管理员统计验证

管理员接口必须携带管理员 JWT。

常用接口：

```text
GET /api/v1/admin/stats
GET /api/v1/admin/reports/summary
GET /api/v1/admin/reports/products
GET /api/v1/admin/reports/orders
GET /api/v1/admin/reports/categories
GET /api/v1/admin/reports/users
```

统计数据必须来自 MySQL 查询或聚合，不要写死。

## 12. 图片上传

当前接口：

```text
POST /api/v1/files/upload
```

生产建议配置：

```text
COS_BUCKET=你的对象存储 bucket
COS_REGION=你的对象存储地域
```

容器本地目录不适合长期保存图片。课程阶段如果暂未开通 COS，需要在答辩中说明图片存储是预留正式结构，后续切换到 COS 或云存储。

## 13. 常见错误

- `MYSQL_ADDRESS` 写成 `localhost`：云托管容器会连自己，不会连到数据库。
- `MYSQL_DATABASE` 写成 `MYSQL_DATABAS`：数据库名读不到。
- 容器监听 `127.0.0.1`：云托管无法转发流量。
- 云托管端口填 5000：本项目正式配置要求填 80。
- 前端仍请求局域网 IP：同学手机不在同一 Wi-Fi 时无法访问。
- `AI_MODE` 仍为 mock：千问接口不会被正式调用。
- `DASHSCOPE_API_KEY` 未配置：AI 接口会返回配置缺失。
- `WECHAT_AUTH_MODE` 仍为 mock：不是真实微信登录。
- `WECHAT_APPID` 或 `WECHAT_SECRET` 缺失：微信 code 无法换 openid。
- MySQL 表没初始化：先执行 `init_db.sql` 或初始化接口。
- token 没携带：检查 `request.js` 是否自动加 `Authorization`。
- 云托管日志出现 500：先看启动日志，再看 `/api/v1/health/db`。
- 图片上传失败：检查上传接口和 COS 配置。
- 管理员初始化失败：检查 `INIT_TOKEN` 和管理员初始化流程。

## 14. 验收清单

完成后至少验证：

1. 手机预览小程序，不需要和电脑在同一个局域网。
2. 微信账号登录成功。
3. 首页商品从云端 MySQL 加载。
4. 发布商品后 MySQL 有记录。
5. AI 标题生成调用真实千问 API。
6. AI 描述润色调用真实千问 API。
7. 收藏写入 MySQL。
8. 创建订单写入 MySQL。
9. 后端模拟支付真实更新订单状态。
10. 聊天消息写入 MySQL。
11. 买家、卖家订单列表可区分显示。
12. 个人资料可修改。
13. 管理员统计来自 MySQL。
14. `/api/v1/health` 正常。
15. `/api/v1/health/db` 正常。
16. 云托管日志无关键启动错误。
